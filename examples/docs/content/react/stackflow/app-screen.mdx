## Usage

```tsx
import {
  AppBar,
  AppBarBackButton,
  AppBarCloseButton,
  AppBarIconButton,
  AppBarLeft,
  AppBarMain,
  AppBarRight,
} from "grape-design/ui/app-bar";
import { AppScreen, AppScreenContent } from "grape-design/ui/app-screen";
```

```tsx
<AppScreen theme="cupertino">
  <AppBar>
    <AppBarLeft>
      <AppBarBackButton />
    </AppBarLeft>
    <AppBarMain>Title</AppBarMain>
    <AppBarRight>{/* actions */}</AppBarRight>
  </AppBar>
  <AppScreenContent>{/* content */}</AppScreenContent>
</AppScreen>
```

## Examples

### Tones

Set `tone="transparent"` on AppScreen to use a transparent background.

- The AppBar background becomes transparent.
- A gradient is shown at the top of the AppScreen (including the mobile OS status bar area).
  - Set the `gradient` prop to `false` to hide it.

### Layer Offset Top

Use the `layerOffsetTop` prop to adjust the top offset of `AppScreenContent`.

When using `tone="transparent"` and `gradient`, you typically set `layerOffsetTop="none"` so the content area extends into the mobile OS status bar area.

<Callout type="info">

Use a `viewport` meta tag that includes `viewport-fit=cover` to handle safe areas (e.g. display cutout / notch) correctly.

```html
<meta
  name="viewport"
  content="width=device-width, initial-scale=1, viewport-fit=cover"
/>
```

</Callout>

### Preventing Swipe Back

Use the `preventSwipeBack` prop to disable the swipe-back gesture on AppScreens with `theme="cupertino"` by not rendering the edge region.

### Transition Styles

Use the `transitionStyle` prop to specify the transition when an AppScreen is pushed to the top or popped from the top.

<Callout type="info">

**Top** means the AppScreen created from an activity where `useActivity().isTop === true`.

</Callout>

If not specified, `transitionStyle` defaults by `theme`:

- `theme="cupertino"`: `slideFromRightIOS`
- `theme="android"`: `fadeFromBottomAndroid`

When the top AppScreen is pushed or popped, non-top AppScreens also play the transition.
